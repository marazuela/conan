## 1. D-015 | Automated Candidate Monitor (2026-04-20)

### Purpose

Close the gap opened by D-013 (pre-edge-only mandate). D-013 said candidates must be archived when the catalyst resolves; the monitor automates detection of resolution events between curator passes so the executive summary cannot carry post-edge stale names for more than one operational cycle (≤3 hours worst case).

### New file: `engine/tools/candidate_monitor.py` (806 lines; modified 2026-04-20 23:21:01)

**Iterates**: every active entry in `candidates/_curated_rationales.json` (i.e. not in the `_archived` bucket and not `_draft: true`).

**Probes, per ticker**:


| Probe     | Source                                                   | Surfaces                                                                                                                                                |
| --------- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `price`   | `yfinance.Ticker(sym).history(period="7d")` daily closes | ±15% single-day move → REVIEW; ±20% single-day → contributes to AUTO_ARCHIVE                                                                            |
| `edgar`   | `data.sec.gov/submissions/CIK{cik:010d}.json`            | Resolution forms `{DEFM14A, PREM14A, SC TO-T, SC TO-I, SC 13E3, 425, S-4, SC 14D9}` + (activist archetype only) `{SC 13D/A, DFAN14A, DEFC14A, DEFA14A}` |
| `news`    | `yfinance.Ticker(sym).news`                              | Archetype-specific keywords (`approval`, `CRL`, `acquired`, `pill adopted`, `settlement`, `verdict`, …)                                                 |
| `openFDA` | `api.fda.gov/drug/drugsfda.json`                         | PDUFA archetype only; scoped to drug-name tokens extracted from the MD and curated entry                                                                |


**CIK resolution fallback**: SEC `company_tickers.json` fetched once and cached to `working/_ticker_to_cik_cache.json`, so candidates whose MD never spelled out CIK (VRDN, VERA) still get EDGAR probes.

- **AUTO_ARCHIVE** — requires ANY of:
  - Resolution-form filing AND catalyst date not far in the future

- **NOOP** — no signals

**Reversibility**: every AUTO_ARCHIVE writes to `working/auto_archive_audit.jsonl` (pre-state snapshot + triggers + MD paths). Undo via `python3 tools/candidate_monitor.py --undo <TICKER>`.

**Lazy-imports** `kill_watch.evaluate_rules()` (D-016).

**Writes**: `working/candidate_monitor_report_<date>.json`.

## 2. D-016 | Kill-Watch DSL (2026-04-20)

### Purpose

Each curated candidate carried a free-text `kill_watch` line. Not machine-checkable. Add a structured rule evaluator so candidate-specific invalidation triggers (e.g. AXSM CRL, RGR Beretta withdraws proxy, VRDN REVEAL-2 miss) fire automatically.

### New file: `engine/tools/kill_watch.py` (566 lines; modified 2026-04-20 23:21:09)

**Six probe kinds**:

```
price_move        | threshold on single-day or 5-day move
edgar_form        | presence of specific form codes in SEC submissions feed
edgar_8k_items    | 8-K Item codes AND (v1.1) keyword body match against primaryDocument
news_keyword      | yfinance news + regex; yahoo_news is an alias
competitor_move   | peer tickers with price/volume thresholds
```

**Rule schema**, stored on each curated entry as `kill_watch_rules: [...]`:

```json
{
  "id": "rpay_board_rejects",
  "description": "...",
  "kind": "news_keyword",
  "params": {"keywords": ["board rejects", "reject offer"]},
  "action": "review | archive | cap_watchlist",
  "severity": "HIGH | MEDIUM | LOW"
}
```

**Integration**: `candidate_monitor.run()` lazy-imports `evaluate_rules(entry, ticker, ctx={"cik": cik})` after the price/EDGAR/news probes. Firings append as `rule:<id>(<severity>)` triggers and escalate the decision to AUTO_ARCHIVE only if the rule's `action == "archive"`.

**v1.1 tightening** — `edgar_8k_items` originally matched on 8-K `items` metadata alone, which caused a false fire on AXSM's April 1 balipodect-acquisition 8-K (it carried Item 8.01 "Other Events", matching the generic "Major Amendment" rule). Fixed by making the evaluator fetch the first **40KB** of the filing's `primaryDocument` from `sec.gov/Archives/edgar/data/...` and require at least one of the rule's keywords to hit the body text. Rules without keywords still fire on item-metadata alone; rules with keywords demand a real body match.

**Translator mode**: `python3 tools/kill_watch.py --translate` grep-matches common kill-watch phrases in `_curated_rationales.json` and drafts starter rules to `working/kill_watch_rules_draft_<date>.json`.

## 3. D-017 | Live Price Refresh on Publish (2026-04-20)

### Purpose

`_curated_rationales.json` contains the curator's anchor price at thesis creation time. By the time the next report publishes (hours to a day later), printed upside/downside percentages drift out of sync. In RPAY's case — after Forager's Apr-17 offer drove the stock +29% — the curator's "$2.60-3.00 (-6% to -18%)" downside was wrong by ~20pp.

### New file: `engine/tools/price_refresh.py` (251 lines; modified 2026-04-20 19:05:08)

**Overlay pattern** — does NOT mutate `_curated_rationales.json`. Writes to `working/_live_prices_overlay.json` with 6-hour TTL and a `_live_refresh` meta block (`refreshed_at`, `tickers_refreshed`, `tickers_stale`).

**Algorithm**:

1. For each ticker in the active book, fetch the last daily close via `yfinance.Ticker(sym).history(period="5d")`, take the latest row.
2. Parse curator's prose price targets with a regex recovering the numeric range from strings like `$215-235 (+17% to +28%) — clean approval`.
3. Swap the `(…%)` percentage group for the same dollar range re-evaluated against the new reference price; preserve the narrative tail verbatim.
4. On yfinance failure: leave original `reference_price` in place but prefix `[STALE — live refresh failed]`. The rest of the price_targets block is untouched.

**Consumer integration**: `report_generator._load_curated_rationales()` prefers the overlay if it is younger than 6 hours — otherwise falls back to the original curated file.

**Invocation**: called by `report_generator.publish_reporting()` before any PDF is composed.

### Smoke test

Publish run on 2026-04-20: `refreshed=5 stale=0`. Exec-summary PDF carries live prices — RPAY $3.95 (was $3.18), VRDN $14.84 (was $18.84), AXSM $189.24 (was $184.19), VERA $42.12 (was $43.72), RGR $42.05 (was $41.98). STALE-path tested by stubbing `_fetch_live_price` to return None — all 5 entries correctly prefixed with `[STALE]`.

## 5. D-020 | Scoring Rubric Backtester (2026-04-20)

### Purpose

Validate whether the 7-dimension scoring rubric actually sorts signals by realized forward return.

### New file: `engine/tools/backtester.py` (610 lines; modified 2026-04-20 20:29:27)

**Algorithm**:

1. Reads `signals/signal_log.json`, joins to rich scanner output for each entry.
2. Bulk downloads yfinance prices for every unique symbol (session-scoped cache).
3. Computes +1d / +5d / +21d / +63d forward close-to-close returns.
4. Aggregates per `(scoring_profile, band)` tuple, where `band ∈ {>=35, 25-34, 15-24}` (historical bands — **NOT** the new 30/20/10 introduced in D-034; see Preserved section of D-034).
5. Applies `thesis_direction` so short signals get negated pct return (a short whose stock fell 5% counts as +5%).
6. **Curator ground-truth join** — attaches `_archived` outcomes to any signal whose ticker overlaps, so calibration report shows both statistical backtest and hand-labelled fate.

**Key outputs**:

- Per-profile calibration table: mean + median forward return, hit rate (% positive), p5/p95 at each horizon, per band.
- Rubric sanity check: **Spearman(raw score, +5d forward return)** per profile. ρ > +0.1 = rubric works directionally; ρ < -0.1 = inverted (alarm).

**Writes**: `working/calibration_<YYYYMMDD>.{json,md}`.

### Smoke test (2026-04-20)

169 scored signals across 133 unique yfinance symbols. Spearman(score, +5d) = **-0.15** (slightly inverted on this small sample). Only `activist_governance` profile present. Interpreted as consistent with noise (4 trading days, 16 signals in immediate band) + potential score-35 bucket being public-event-driven (tender-offer filings) where the reaction already happened pre-scan. Flagged for event-study follow-up.

```python
summarize_deal_desk(window_days=14, max_items=12) -> list

```

**US-issuer gate**: `mic ∈ {XNYS, XNAS, XASE, BATS, ARCX, IEXG}`. Non-US → `{status: "unavailable", reason: "non_us_issuer"}` with a human-readable note pointing the curator to native-jurisdiction filings.

**Online mode**:

1. Resolves ticker → CIK via `www.sec.gov/files/company_tickers.json`.
2. Pulls `data.sec.gov/api/xbrl/companyfacts/CIK{cik:010d}.json`.
3. Computes trajectory / variance / EV/EBITDA / deterioration (see below).
4. Writes to `engine/working/deal_context_cache.json` (**30-day TTL**).

### Tier frameworks

#### Trajectory

Up to 3 fiscal years of annual (10-K) XBRL facts:

- **Revenue** — concept preference list tolerates filer variance (`Revenues`, `RevenueFromContractWithCustomerExcludingAssessedTax`, `SalesRevenueNet`).
- **Operating income** — `OperatingIncomeLoss`.
- **COGS** — `CostOfGoodsAndServicesSold`, `CostOfGoodsSold`.
- **SG&A** — `SellingGeneralAndAdministrativeExpense`.
- **Operating cash flow** — `NetCashProvidedByUsedInOperatingActivities`.
- **Capex** — `PaymentsToAcquirePropertyPlantAndEquipment`.
- **FCF** (derived) = OCF − Capex.
- **EBITDA** (derived) = OperatingIncome + D&A.

#### Variance

Pairwise YoY across the trajectory:

- YoY revenue % change.
- Op-margin delta (bps).
- SG&A/Rev delta (bps).
- FCF % change.

#### EV/EBITDA proxy

`EV = market_cap (shares × ref_price) + long_term_debt + current_portion − cash_and_equivalents`.
`EV / EBITDA`. Emits `insufficient_data` gracefully when any ingredient is missing.

#### Financial deterioration score (activist lens)

```
score = -op_margin_delta_bps + sga_pct_delta_bps     # saturated at 100
```

Color bands: GREEN ≤25, YELLOW 26-50, ORANGE 51-75, RED >75. Captures the "margin compression + SG&A bloat" pattern activists target.

## 20. D-035 | End-to-End Thesis Analyst (9th hook, 2026-04-21)

### The failure being structurally prevented

D-033's auto-curation produced templated prose keyed on profile+direction, not on the actual filing. Pathological mislabel: `6027` (弁護士ドットコム) got the `activist_governance_composed` boilerplate about "activist campaign / shareholder stake / proxy contest" when the underlying TDNet was a ¥201M impairment disclosure with **no** activist dimension.

Pedro's directive: *"We need an end to end tool from data ingestion, classification of signals, identification of most promising candidates which would be included in dashboard and executive summary, and for these selected candidates the tool should execute an analysis for a potential investment thesis. If we don't do the process end to end then it does not make any sense."*

### New file: `engine/tools/thesis_analyst.py` (1207 lines; modified 2026-04-21 04:29:39)

**Two-stage, evidence-first pipeline**.

#### Stage 1 — Fact pack assembly (mechanical, in-engine)

1. **Source doc fetch** — TDNet / EDINET / SEC 8-K / LSE RNS / etc. Cached under `engine/working/source_doc_cache/` with descriptive filenames.
2. **Text extraction** — first ~600 chars of raw text (PDF / HTML / text as applicable).
3. **Disclosure tag pattern match** — 17-tag taxonomy:

```
impairment_disclosure, guidance_revision, board_change, tender_offer,
secondary_offering, mna_approach, fda_submission, pdufa_outcome,
going_concern, restatement, short_seller_report, activist_stake,
insider_txn, buyback, dividend_cut, settlement, consent_decree
```

1. **Numeric parsing** — yen/USD amounts, percentages, dates.
2. **Prior signal lookback** — 180-day window keyed on issuer.
3. **Market context snapshot** via yfinance:

```
last           # latest close
d5             # 5-day return
d30            # 30-day return
d90            # 90-day return
vol_ann        # annualised realised volatility
adv_30d        # 30-day average daily dollar volume
```

1. **Enrichment snapshots** — any biotech / legal / deal_context enrichment already on the signal.

Persists each pack to `engine/working/fact_packs/{TICKER}_{YYYY-MM-DD}.json`.

#### Stage 2 — Deep thesis synthesis (evidence-only, no generic prose)

Reads the fact pack, writes `deep_thesis` block:

```
schema_version    : "1.0"
confidence        : HIGH | MEDIUM | LOW   (source-doc availability × prior-signal convergence × enrichment presence)
directional_view  : LONG | SHORT          (disclosure-tag polarity + price trajectory)
situation         : <prose, evidence-anchored — grounded in parsed tags + numerics + source excerpt>
evidence_bulls    : [<concrete cites — price momentum, volatility, enrichment tier, disclosure polarity>]
evidence_bears    : [<concrete cites>]
open_questions    : [<handoff to downstream deep research where mechanical analyst can't reach>]
disclosure_tags   : [<matched taxonomy>]
extracted_numerics: {yen_amount_raw, usd_amount, pct_change, ...}
market_context    : {last, d5, d30, d90, vol_ann, adv_30d}
```

**No boilerplate**. If a field has no evidence, it is omitted.

### Public API

```python
thesis_analyst.run(force=False, only_ticker=None) -> dict   # idempotent
```

- `--force` re-analyses even schema_version-stamped candidates.
- `--only-ticker TKR` targets a single candidate.
